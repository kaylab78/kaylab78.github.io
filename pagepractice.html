Starting an HTML and CSS Project

Set up your files and file structure

  1. Create a new subfolder in your "projects" folder.
  2. Name it the title of your project. (Use an underscore instead of a space, if necessary.)
  3. Create a subfolder inside your new folder, and name it "resources."
  4. Create a subfolder inside of "resources," and name it "css."
  5. Open a new file in Visual Studio Code, and name it "index.html".
  6. Open a new file in Visual Studio Code, and name it "style.css".
  7. Copy and paste the following code to set up your HTML document:

    <!DOCTYPE html>
    <html>
    <head>
      <link href="./resources/css/style.css" type="text/css" rel="stylesheet">
      <title>Project Title</title>
    </head>
    <body>
    </body>
    </html>

  8. Add any other metadata within the head tags.
  9. Begin coding within the body tags.

General: HTML Tags, Attributes, and Keywords

Tag / Attribute / Keyword Definition / Description
& l t ; In order to show HTML tags as text, use these symbols together to represent <. Remember it as "less-than."
& g t ; In order to show HTML tags as text, use these symbols together to represent >. Remember it as "greater-than."
& n b s p ; Use these symbols together to insert extra space on the same line. Remember it as "no break space."
<a></a> Anchor element, links to other webpages.
Example on linking text: <a href="https://website.com">Text</a>
Example on linking an image: <a href="https://website.com"><img src="photo.jpg /></a>
Example on linking to the same page using a target: <p id="top">Text</p>   <a href="#top">Top of page</a>
Absolute Path Full URL stored in a different folder.
alt Alt attribute, alternative text, metadata, description of the image, etc.
<article></article> Content that makes sense on its own.
<aside></aside> Mark additional information that can enhance another element but isn't required in order to understand the main content (bibliography, endnotes, comments, pullquotes, editorial sidebars, additional info)
Attribute Content added to the opening tage of an element; used in different ways from providing info to changing styling. Ex: <div id="info"></div>
<audio></audio> Embeds audio.
<audio>
 <source src=""link"" type=""audio/mp3"">
</audio>
<body></body> Content inside the body tags is the only content that will appear on the screen.
<br> Line break (Only a starting tag)
<!-- Comment --> Characters inbetween these symbols are ignored by browser. Comments are helpful to understand code later, and they allow you to experiment with new code without deleting old code.
<div></div> Division / A container that divides the page into sections. Useful for grouping together elements in HTML.
<!DOCTYPE html> Document type declaration
<em></em> Styles text as italic.
<embed> Self-closing tag. Embed any media content, can use local files or a link from another website.
Family Tree In HTML structure, an element inside another element is the "child" element nested inside the "parent" element. HTML hierarchy is important because child elements can inherit behavior and styling from their parent element.
<figcaption></figcaption> Caption, describes media. Goes inside <figure> and moves with <figure>.
<figure></figure> Used to encapsulate media such as an image, illustration, etc. Referenced in the main flow of the document.
<footer></footer> Bottom of content (contact, copyright, terms, site, etc.)
<h1></h1> Main heading
<h2> - <h6> Subheadings that diminish in size the greater the number. (Opening and closing tags)
<head></head> Contains metadata of a webpage.
<header></header> Container for navigational links or introductory content containing <h1> to <h6> tags.
href Hyperlink reference (see <a></a>)
HTML Hypertext Markup Language
<html></html> HTML structure after doctype
<img> Image, self-closing tag. Ex: <img src="image-location.jpg" />
Indentation Use space and tab to easily visualize which elements are nested within other elements. (2 spaces of indentation)
<li></li> List items, use within an unordered or ordered list.
<main></main> Encapsulates dominant content. Screen readers identify stuff in this tag as the bulk of the content.
<nav></nav> Defines a block of navigation links (menus, ToC). Can be used inside header or on its own.
<ol></ol> Ordered list, numbered items
<p></p> Paragraphs contain a block of plain text.
Relative Path Filename that shows the path to a local file.
Root Directory Main folder where all files for the project are stored.
<section></section> Inside body and defines areas with the same theme (chapters, headings, etc.)
<span></span> Contains short pieces of text or other HTML; used to separate small pieces of content that are on the same line as other content.
src Source attribute, location of a media file
<strong></strong> Styles text as bold.
target Target attribute specifies how a link opens.
Example on opening a new window: <a href="https://website.com" target="_blank">Text</a>
<title></title> Page title is displayed on a browser tab.
<ul></ul> Unordered list, bullet points
<video></video> Video tags.
Ex: <video src="video.mp4" width="320" height="240" controls>Video not supported</video>
Whitespace A tool used to make code easier to read and follow. Put elements on their own lines.

Tables: HTML Tags, and Attributes

Tag / Attribute Defintion / Description
colspan Spanning columns attribute, tells us how many columns (cells) a table data element encompases.
rowspan Spanning rows attribute, tells us how many rows a table data element encompases.
scope Scope attribute declares a heading for a row or column within a <th>.
<table></table> Encloses an entire table.
<tbody></tbody> Should contain all of the table's data excluding the table headings.
<td></td> Table data element. Each <td> is a cell and creates columns within a <tr>
<tfoot></tfoot> Table footer, bottom part of a long table (Ex: totals, sums, differences, etc.)
<th></th> Table heading, a title for a row and placed within a table row
<thead></thead> Table heading contained inside this element for both columns and rows.
<tr></tr> Table rows

Tag / Attribute / Keyword Definition / Description
action Where <form> information is sent
Client-side Validation Check data on the browser, occurs before data is sent to the server
<form></form> Requires the location of where the info goes and what HTTP request to make.
<form>
 <label for="text">Dropdown</label>
 <select id="text" name="text">
  <option value="text">Text</option>
 </select>
</form>		 Dropdown List
<input pattern="[0,9] {14,16}"> Regular Expression (regex), a sequence of characters that make up a search pattern.
<input required> Requiring an input
<input type="checkbox"> Checkbox input, need individual inputs for each checkbox.
<input type="number"> Number input
<input type="number" min="1" max="100"> Setting a minimum and maximum for a number or range type.
<input type="password"> Password input
<input type="radio"> Radio button input, only one can be selected.
<input type="range"> Range input (slider). Ex: <input type="range" min="0" max="100" step="10">
<input type="submit" value="send"> Submit form
<input type="text"> Form text input, type determines how the data is rendered on the webpage and what data the input field accepts.
<input type="text" minlength="5" maxlength="40"> Checking text length with a minimum and maximum number of characters
<input type="text">
<datalist id="text">
 <option value="text"></option>
</datalist>		 Datalist input
<label></label> Used to label an <input> element. To associate with an <input>, use an id attribute
method Assigned an HTTP verb included in the HTTP request, see <form>.
Server-side Validation When data is sent to another machine for validation
<textarea></textarea> Textarea element, bigger text field. Text between the opening and closing tags will be pre-filled text.
Validation Concept of checking user provided data against required data.
value Can use a value attribute to pre-fill a form. Otherwise, whatever is typed into the input element becomes the value attribute.
